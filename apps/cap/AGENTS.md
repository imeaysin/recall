# Cap landing UI

Frontend-only Next.js 16 copy of Cap's complete landing experience.

- Preserve the source landing page's sections, responsive behavior, animations,
  typography, and assets.
- Do not add API routes, authentication, database code, storage, workflows, or
  `@cap/*` packages.
- Build controls and primitives with `@workspace/ui-shadcn`.
- Keep Cap-specific visual tokens and compatibility styling local to this app.
